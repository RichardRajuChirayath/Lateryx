Installation

Option 1: GitHub Action (Recommended)

Add Lateryx to your CI/CD pipeline:

- uses: RichardRajuChirayath/Lateryx@v2
  with:
    terraform_directory: './infrastructure'
    fail_on_breach: 'true'
    severity_threshold: 'HIGH'

Option 2: Local Installation

# Clone the repository
git clone https://github.com/RichardRajuChirayath/Lateryx.git
cd Lateryx

# Install as a package
pip install -e .

# Verify installation
lateryx --version

Option 3: Docker

docker pull ghcr.io/richardrajuchirayath/lateryx:latest
docker run -v $(pwd):/workspace lateryx scan /workspace/infrastructure

GitHub Action

The GitHub Action is the recommended way to use Lateryx. It automatically scans every Pull Request and posts detailed security reports.

Full Configuration

name: Security Scan

on:
  pull_request:
    paths:
      - 'infrastructure/**'
      - 'terraform/**'

jobs:
  lateryx:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      
      - uses: RichardRajuChirayath/Lateryx@v2
        with:
          terraform_directory: './infrastructure'
          fail_on_breach: 'true'
          severity_threshold: 'HIGH'  # CRITICAL, HIGH, MEDIUM, LOW
          output_format: 'both'       # json, markdown, both
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}

Inputs

Input Default Description
terraform_directory . Path to Terraform files
fail_on_breach true Fail the workflow if issues are found
severity_threshold HIGH Minimum severity to fail on
output_format both Output format (json, markdown, both)

CLI Scanner

Use the CLI for local development and scripting:

# Basic scan
lateryx scan ./infrastructure

# JSON output for CI/CD
lateryx scan ./infrastructure --format json

# Fail on issues (for scripts)
lateryx scan ./infrastructure --fail-on-breach --severity CRITICAL

Auto-Remediation

Lateryx can automatically generate fixes for common security issues:

High-Confidence Fixes: Lateryx only auto-applies fixes with 80%+ confidence. Lower-confidence fixes are suggested for manual review. 
from src.auto_remediation import get_auto_remediator

remediator = get_auto_remediator()
patches = remediator.analyze_file("main.tf", content)

for patch in patches:
    print(f"Fix: {patch.description}")
    print(f"Confidence: {patch.confidence * 100}%")

Kubernetes Security

Scan Kubernetes manifests for security issues:

from src.kubernetes import get_kubernetes_scanner

scanner = get_kubernetes_scanner()
resources, findings = scanner.scan_directory("./k8s/")

for finding in findings:
    print(f"[{finding['severity']}] {finding['description']}")

Detected Issues

Configuration File

Create a lateryx.config.yml in your repository root:

version: "1.0"

# Define your most critical resources
crown_jewels:
  name_patterns:
    - "*production*"
    - "*customer*"
    - "*payment*"
  types:
    - aws_rds_instance
    - aws_s3_bucket

# Customize risk scoring
risk_weights:
  public_exposure: 1.0
  sensitive_data: 0.9
  admin_access: 0.8

# Severity thresholds
thresholds:
  fail_on: HIGH
  warn_on: MEDIUM
